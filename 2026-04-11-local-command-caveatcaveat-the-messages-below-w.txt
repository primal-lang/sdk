10. MEDIUM: usedParameters tracking for let bindings unclear

What is wrong: The semantic analyzer tracks used parameters via usedParameters
set (semantic_analyzer.dart:68, 248). When a let binding is used in the body,
it should be tracked to avoid false "unused parameter" warnings.

Why it matters: The document says at line 283-284:
"The usedParameters tracking for UnusedParameterWarning works correctly"

But it doesn't explain how. Let binding names are added to
availableParameters. When the body references a let binding, the current
_checkIdentifierExpression would add it to usedParameters. But let bindings
aren't function parameters — we shouldn't report them as "unused parameters".

The algorithm needs to differentiate:
- Function parameters: track usage, warn if unused
- Let bindings: don't warn (or warn separately with a different warning type)

Consequence: Without modification, let bindings would be incorrectly tracked
alongside parameters, potentially causing incorrect warnings.

File references:
- Design doc lines 270-284
- lib/compiler/semantic/semantic_analyzer.dart:68, 79-88, 248
